HTML Tags

The Grammar

The grammar.bnf is a recursive grammar for matching HTML tags. The match requires that the name in the end tag match the name in the start tag. This is accomplished with the “parent backreference” operator \%p.

The Input

The file HMTL.txt is the input. It contains nested HTML tags. Paste it into the Input panel and click Parse Input. Select the P rule name from the drop-down box. Notice that there are 3 matches to this rule. The first match is the entire string because it is the outer-most HTML tag. However, click the forward arrow, ⏵, and watch it highlight the inner tags.

To convince yourself that this grammar really only matches tags with matching names in the end tag, go back to the Input panel and edit the last tag from </div> to, say, </div2> and click Parse Input again. Now you will see the State panel open to indicate that the parse failed.
